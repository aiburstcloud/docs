Two burst modes

Choose your default. Override per-request.

edge_burst

Local GPU primary, cloud overflow
User → Local GPU (primary, free)      ↘ Cloud GPU (burst when overloaded)

Best for cost optimization. Data stays local by default. Cloud is the safety valve.

cloud_burst

Cloud primary, local for sensitive data
User → Cloud GPU (primary, fast)      ↘ Local GPU (sensitive data only)

Best for performance-first teams that need sovereignty-on-demand for regulated data.

Three-axis routing engine

🔒

Data Sovereignty

Sensitive keywords detected? Always routed to your local GPU. In both modes. No exceptions.

💰

Cost Control

Set a daily cloud budget. When it's exhausted, requests fall back to local. No surprise bills. Ever.

⚡

Latency Optimization

Primary backend overloaded? Automatically burst to the secondary. Sub-second failover.
